**Good user story characteristics:**

**Independent:** Can be built without depending on other stories
```
Good: "As a user, I want to view my order history"
Bad: "As a user, I want to view my order history on the dashboard"
(This creates dependency on dashboard story)
```

**Valuable:** Delivers clear benefit to users
```
Good: "As a customer, I want to save items to a wishlist so I can purchase them later"
Bad: "As a developer, I want to refactor the database schema"
(Second one has technical value but not direct user value)
```

**Testable:** Clear criteria for when it is done
```
Good: "As a user, I want to receive email confirmation after registration"
Bad: "As a user, I want a better registration experience"
(Second one is too vague to test)
```

**Small:** Can be completed in one sprint
```
Good: "As a customer, I want to add items to my cart"
Too Large (Epic): "As a customer, I want a complete e-commerce checkout experience"
(Break the epic into smaller stories)
```
