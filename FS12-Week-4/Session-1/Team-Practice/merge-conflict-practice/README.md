# Merge Conflict Practice

A hands-on guide to practicing merge conflicts with a partner (or solo) using real collaboration workflows and feature-based scenarios.

## Quick Start

1. Read the complete guide: [MERGE-CONFLICT-GUIDE.md](MERGE-CONFLICT-GUIDE.md)
2. Find a partner to collaborate with (or practice solo using branches)
3. Follow the step-by-step instructions with real-world feature scenarios
4. Practice resolving conflicts together, including multiline conflicts

## Prerequisites

Before starting, complete these Codecademy courses:
- [Learn Git](https://www.codecademy.com/learn/learn-git)
- [Learn Git Branching and Collaboration](https://www.codecademy.com/learn/learn-git-branching-and-collaboration)

## What's Included

- **MERGE-CONFLICT-GUIDE.md** - Complete step-by-step instructions with real-world context
- **starter-files/** - HTML, CSS, and JavaScript files for practice
- **tests/** - Automated tests to verify your code after resolving conflicts

## What You'll Practice

- **Simple Single-Line Conflicts** - Basic greeting and styling changes
- **Multiline Function Conflicts** - Feature enhancements that span multiple lines
- **Complex Configuration Conflicts** - Object property merging scenarios
- **Real-World Feature Scenarios** - Dark mode themes, SEO updates, analytics integration
- **Solo Practice Option** - Use branches to simulate collaboration if you don't have a partner

## Testing Your Code

After resolving conflicts, always run:

```bash
npm test
```

This ensures your code works correctly after resolution.
