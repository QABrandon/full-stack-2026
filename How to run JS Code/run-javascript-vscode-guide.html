🚀 Running JavaScript in VS Code

A step-by-step student guide — from console.log("Hello, Codecademy") to running in seconds

🎯 Goal: By the end of this guide you will be able to type a line of JavaScript and see its output instantly inside VS Code — using whichever method works best for you.
⭐ Start Here

Method 1 — Run & Debug Panel (built-in, no extensions needed)

VS Code has a built-in Run and Debug panel that can run a JavaScript file directly using Node.js — no extensions, no terminal commands. It also lets you set breakpoints and step through your code line by line when you're ready to debug.

⚠️ Node.js must be installed and available in your PATH for this to work. See Method 2 for how to install Node.js, and Method 5 for Mac and Windows PATH setup instructions.

Step-by-step

  1. Create a file called hello.js and add your code:
    console.log("Hello, Codecademy");
    Save the file with Ctrl+S / Cmd+S.
  2. Open the Run and Debug panel — click the bug-with-play-button icon in the left sidebar, or press Ctrl+Shift+D (Windows/Linux) / Cmd+Shift+D (Mac).
  3. At the top of the panel you'll see a blue "Run and Debug" button. Click it.
    Alternatively, press F5 from anywhere in the editor while your .js file is open.
  4. VS Code will ask you to select a debugger. Choose Node.js from the dropdown list.
  5. Your file runs immediately. The output appears in the DEBUG CONSOLE panel at the bottom:
    debug console Hello, Codecademy

Bonus: setting breakpoints

Click in the grey margin to the left of any line number to add a red dot — a breakpoint. When you run with F5, execution will pause at that line so you can inspect variables. This is handy once you start writing longer scripts and need to see what's happening inside your code.

💡 Tip: If VS Code asks you to create a launch.json file, you can safely ignore it for simple scripts — just pick Node.js and your file runs directly without any configuration file needed.
Reliable

Method 2 — Run with Node.js in the VS Code Terminal

This is the most reliable and universally recommended approach. Node.js lets you run JavaScript files directly from your terminal — no browser needed.

Step 1 – Install Node.js (skip if already installed)

  1. Go to nodejs.org and download the LTS version for your operating system.
  2. Run the installer and accept all defaults.
  3. Verify it worked. Open the VS Code terminal (Ctrl + ` on Windows/Linux, Cmd + ` on Mac) and type:
    node --version
    You should see something like v20.x.x. If you do, Node is ready.

Step 2 – Create and run your first file

  1. In VS Code, open a folder (File → Open Folder) or create a new one.
  2. Create a new file. Click the New File icon in the Explorer panel and name it hello.js.
  3. Type the following into hello.js:
    console.log("Hello, Codecademy");
  4. Save the file (Ctrl+S / Cmd+S).
  5. Open the terminal in VS Code (Ctrl + `) and run:
    node hello.js
  6. You should see:
    output Hello, Codecademy
✅ It works! You can now edit hello.js, add more console.log lines, and re-run with node hello.js any time.
Common error: "node is not recognized as a command" — this means Node.js was not installed or VS Code needs to be restarted after installation. Close and reopen VS Code, then try again.
Easy

Method 3 — Code Runner Extension (one-click run button)

The Code Runner extension adds a ▶ Run button to VS Code. It uses Node.js under the hood but saves you typing in the terminal every time.

  1. Make sure Node.js is installed first (see Method 2, Step 1).
  2. In VS Code, click the Extensions icon in the left sidebar (or press Ctrl+Shift+X).
  3. Search for Code Runner and install the one by Jun Han (it has millions of downloads).
  4. Create or open your hello.js file with console.log("Hello, Codecademy").
  5. Run it in one of three ways:
    • Click the ▶ Run Code button in the top-right corner of the editor
    • Right-click in the file → Run Code
    • Press Ctrl+Alt+N (Windows/Linux) or Ctrl+Option+N (Mac)
  6. The output appears in the OUTPUT panel at the bottom:
    output Hello, Codecademy
Note: Code Runner's OUTPUT panel is read-only. If your code uses prompt() or needs user input, use Method 2 (terminal) instead.
Popular

Method 4 — Quokka.js (live inline output)

Quokka.js shows output right next to your code as you type — great for experimenting and learning. The free "Community" version covers most student needs.

Troubleshooting common Quokka problems:

⚠️ Quokka requires Node.js. If Quokka shows a red error or nothing happens, install Node.js first (Method 2, Step 1), then restart VS Code.
  1. Ensure Node.js is installed (see Method 2).
  2. Install the Quokka.js extension: press Ctrl+Shift+X, search Quokka.js, and install.
  3. Start Quokka on a file two ways:
    • New scratch file: Press Ctrl+Shift+P, type Quokka: New JavaScript File, hit Enter
    • Existing file: Open hello.js, then press Ctrl+Shift+P → Quokka: Start on Current File
  4. Type your code:
    console.log("Hello, Codecademy"); // ← Quokka shows output here inline
  5. You will see the result appear right in the editor next to the line:
    quokka Hello, Codecademy

Quokka error troubleshooting:

  • Red squiggle / "Cannot find module" — run npm init -y in the terminal inside your folder, then restart Quokka.
  • "Quokka license required" — use the free Community version: press Ctrl+Shift+P → Quokka: Use Community.
  • Nothing appears — make sure the file extension is .js (not .txt) and Node.js is installed.
Extension

Method 5 — JavaScript REPL VS Code Extension

The JavaScript REPL extension by achil brings an interactive scratchpad directly inside VS Code — run any JavaScript expression and see results inline without leaving your editor.

Extension ID: achil.vscode-javascript-repl

⚠️ Requires Node.js version 10.11.0 or higher, and the node executable must be available in your system PATH. See below for how to set this up on Mac and Windows.

How to make sure Node is in your PATH

🍎 Mac

  1. Download and run the Node.js LTS installer from nodejs.org. The Mac installer automatically adds node to your PATH — no extra steps needed.
  2. Verify it worked. Open Terminal (search Spotlight for "Terminal") and run:
    node --version
    You should see a version number like v20.x.x.
  3. If you see "command not found", Node may have been installed via a version manager. Run this to check where it lives:
    which node
    Then add that path to your shell profile. For most Macs (zsh), open Terminal and run:
    echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
    Then restart VS Code.

🪟 Windows

  1. Download and run the Node.js LTS installer from nodejs.org. During install, make sure the option "Add to PATH" is checked (it is by default).
  2. After installing, fully close and reopen VS Code — this is required for VS Code to pick up the new PATH.
  3. Verify in the VS Code terminal (Ctrl + `):
    node --version
    You should see v20.x.x or similar.
  4. If you see "node is not recognized", add Node manually:
    • Search Windows for "Environment Variables" → click Edit the system environment variables
    • Click Environment Variables → find Path under System variables → click Edit
    • Click New and add: C:\Program Files\nodejs\
    • Click OK on all dialogs, then restart VS Code.

Installing and using the JavaScript REPL extension

  1. In VS Code, press Ctrl+Shift+X to open Extensions, then search for achil.vscode-javascript-repl. Click Install.
    You can also paste the extension ID directly into the search box for an exact match.
  2. Create or open any .js file, or open a new untitled file and set the language to JavaScript.
  3. Type your code:
    console.log("Hello, Codecademy");
  4. The REPL evaluates your code as you type and shows the result inline in the editor:
    repl Hello, Codecademy
  5. To exit the REPL or stop evaluation, close the file or switch to a non-JS file.
Still getting errors after installing Node? The most common fix is to fully close and reopen VS Code after installing Node.js — VS Code only reads PATH when it first launches.
No Node needed

Method 6 — Link an external JS file to HTML & open in the browser

This approach requires zero installs — every computer already has a web browser that can run JavaScript. You keep your JavaScript in its own .js file, link it from an HTML file, open the HTML in Chrome/Firefox/Edge, and see the output in the browser's built-in Developer Console. This is also how real web projects are structured.

✅ No Node.js required. This is the only method on this list that works without installing anything extra.

Step 1 — Create your two files

You need both files in the same folder.

  1. In VS Code, open a folder (File → Open Folder). Create a new file called hello.js and add your JavaScript:
    console.log("Hello, Codecademy");
    Save the file.
  2. Create a second file in the same folder called index.html. Paste in the following — notice the <script src="hello.js"> tag at the bottom, which loads your external JS file:
    <!DOCTYPE html> <html> <head> <title>Hello</title> </head> <body> <!-- This loads hello.js from the same folder --> <script src="hello.js"></script> </body> </html>
    Save this file too.

Step 2 — Open in the browser

  1. Open index.html in your browser. Two easy ways:
    • Drag and drop index.html from your file explorer into a browser window
    • Right-click index.html in VS Code's Explorer panel → Reveal in File Explorer / Finder, then double-click it
  2. The page will look blank — that's normal. console.log writes to the Developer Console, not the visible page. Open it with:
    • Windows/Linux: F12 or Ctrl+Shift+J
    • Mac: Cmd+Option+J
    Then click the Console tab.
  3. You should see your output:
    browser console Hello, Codecademy
  4. Each time you edit hello.js and save, refresh the browser (F5 or Ctrl+R) to re-run the code.

Bonus: use Live Server for auto-refresh

Install the Live Server extension by Ritwick Dey (ritwickdey.liveserver) from the Extensions panel. Then right-click index.html in VS Code → Open with Live Server. The browser will automatically refresh every time you save either file — no manual reloading needed.

Important: Both index.html and hello.js must be in the same folder, otherwise the browser can't find the script. Also remember: console.log output is invisible on the page itself — always open the Developer Console (F12) to see it.

Method Best for… Needs install? Beginner friendly?

💡 Recommendation for students: If you can't install Node.js, start with Method 6 — just a browser and VS Code. Otherwise, start with Method 1 (Run & Debug Panel) — built into VS Code, no extra extensions needed.

🔧 General Troubleshooting

  1. "command not found: node" — Install Node.js from nodejs.org, then fully close and reopen VS Code.
  2. Nothing happens when I run my file — Make sure your file is saved (Ctrl+S) and ends in .js.
  3. Output shows an error, not my text — Check your code carefully. Common mistakes: missing quotes around text, missing parentheses after log, or a typo in console.
  4. VS Code terminal won't open — Go to View → Terminal from the menu bar.
  5. Still stuck? — Copy the exact error message and search for it on Stack Overflow or ask your instructor.
